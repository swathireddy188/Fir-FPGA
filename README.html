Parameterized FIR Filter — FPGA (SystemVerilog)

A streaming, synthesizable FIR digital filter for Xilinx Artix-7 / Vivado, with a self-checking Questa/ModelSim testbench and a Python golden reference model. The canonical FPGA-DSP building block — RTL design, verification, and DSP fundamentals end to end.

What an FIR filter does

It shapes a stream of samples by taking a weighted sum of the current and previous inputs:

y[n] = c0*x[n] + c1*x[n-1] + ... + c(N-1)*x[n-(N-1)]

The coefficients define the response (low-pass, high-pass...). Change only the coefficients and the same hardware becomes a different filter.

Design

  • Parameterized: DATA_WIDTH, COEFF_WIDTH, N_TAPS, coefficients.
  • Fully signed, full-precision output — matches the golden model bit-for-bit.
  • Streaming: assert in_valid with a sample, get out_valid + out_data one clock later.
  • Default 8-tap smoother: {1,2,3,4,4,3,2,1}.

Verify it (Questa)

python3 model/fir_golden.py          # print verified expected values
vlog rtl/fir_filter.sv tb/fir_filter_tb.sv
vsim -c work.fir_filter_tb -do "run -all; quit"

Or in Vivado xsim:

xvlog -sv rtl/fir_filter.sv tb/fir_filter_tb.sv
xelab fir_filter_tb -R
Expect every sample to print OK and the summary RESULT: PASS. The impulse produces the coefficients scaled by 50 (50,100,150,200,200,150,100,50) — instant visual proof the filter is correct.

Synthesize (Vivado)

  • Target your Artix-7 part (e.g. xc7a35t for Arty A7).
  • Add the RTL and the XDC; fill in board-specific pin LOCs.
  • Run synthesis + implementation; check the timing report for positive WNS — your timing-closure proof, same metric as your UART project.

Swap in a real filter (DSP extension)

MATLAB:  b = fir1(7, 0.25);  coeffs = round(b * 64);
Python:  coeffs = np.round(firwin(8, 0.25) * 64)

Put the integer coefficients in both fir_golden.py and the RTL COEFFS parameter, re-run the model to refresh expected values, re-simulate. The hardware never changes — only the numbers.
